# 🍅 Tomato – Food Delivery Web Application

A modern food ordering web application built with **React, Vite, and Firebase**.  
The project demonstrates modular frontend architecture, authentication workflows, state management, and responsive UI development in a production-style environment.

🔗 **Live Demo:** https://tomato-food-app-alpha.vercel.app  
📂 **Repository:** https://github.com/muhamedismaili/TomatoFoodApp  

---

## 🚀 Overview

Tomato is a responsive food delivery platform that allows users to:

- Browse categorized meals  
- Add and remove items from a shopping cart  
- View dynamic cart totals  
- Register and authenticate securely  
- Complete a structured checkout process  

The application simulates a real-world e-commerce workflow and emphasizes clean component design and scalable frontend architecture.

## ✨ Key Features

- Component-based architecture  
- Client-side routing with React Router  
- Authentication (Login / Signup)  
- Dynamic cart logic with real-time total calculation  
- Category-based product filtering  
- Structured checkout form handling  
- Responsive layout across screen sizes  
- Optimized production build using Vite  

---

## 🧠 Architecture

The application follows a modular React structure:

- Reusable UI components  
- Centralized state handling for cart management  
- Separation of page-level views using routing  
- Authentication state-based conditional rendering  
- Clear separation between presentation and business logic  

This structure supports scalability and allows easy extension to API-driven backend systems.

## 📦 Installation & Setup

Clone the repository:

```bash
git clone https://github.com/muhamedismaili/TomatoFoodApp.git
cd TomatoFoodApp
```

Install dependencies:

```bash
npm install
```

Run development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```
